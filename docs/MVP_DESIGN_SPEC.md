## 1. 제품 개요

### 1.1 한 문장 설명

가족과 가까운 교회 구성원이 계정 없이 바로 열어 반 아이들의 주간 출석과 큐티 여부를 빠르게 기록하고, 간단한 생일·출석·큐티 현황을 확인하는 모바일 우선 웹앱이다.

### 1.2 이번 MVP의 성격

이번 MVP는 공개 SaaS가 아니라 **패밀리 오픈**이다.

- 로그인, 초대, 역할 권한, 결제는 만들지 않는다.
- 한 교회 또는 한 가족 규모의 사용을 전제로 한다.
- Supabase Postgres의 정규화 업무 테이블을 사용해 여러 기기에서 같은 데이터를 공유한다.
- 로그인 없는 패밀리 오픈이므로 publishable key 기반 공개 읽기/쓰기를 허용한다.
- 앱 상태는 브라우저 `localStorage`에 저장하지 않고 Supabase DB를 기준으로 읽고 쓴다.
- 앱 런타임은 테스트용 샘플 반/아이 데이터를 초기값이나 빈 DB fallback으로 사용하지 않는다.

### 1.3 성공 기준

- 휴대폰에서 3단계 이내로 출석 체크 화면에 도달한다.
- 아이별 출석과 큐티를 한 손으로 빠르게 바꿀 수 있다.
- 저장 여부가 명확하고 새로고침·기기 변경 후에도 Supabase에 저장된 데이터가 유지된다.
- 아이 상세 정보와 보호자 연락처는 Supabase 정규화 테이블에 저장된다.
- 모바일 390px 화면에서 가로 스크롤 없이 사용할 수 있다.

## 2. MVP 범위

### 2.1 Must Have

1. 로그인 없는 홈 진입
2. 기본 반과 아이 목록
3. 아이 추가
4. 반별 출석/결석 체크
5. 아이별 큐티 완료 체크
6. 간단한 주간 메모
7. Supabase 원격 저장과 새로고침 후 유지
8. 이번 주 출석 요약
9. 이번 달 큐티 참여 아동 수와 완료 건수
10. 이번 달 생일자 목록
11. 모바일 우선 하단 내비게이션
12. 기본 빈 상태, 저장 상태, 오류 방어
13. 아이 상세 정보 입력: 이름, 사진, 성별, 생년월일, 보호자 여러 명, 주소, 이메일, 등록일, 반, 특이사항
14. 아이 목록 반별/이름 필터와 가나다/반별 정렬
15. 통계 카드 상세 명단 모달과 명단 복사
16. 선생님 등록: 이름, 생일, 반, 전화번호, 사진
17. 설정 화면 반 등록: 반 이름과 담임 선생님 선택
18. 홈과 출석 반 선택에 담임 선생님 표시
19. 전도사님 공유 메모 체크
20. lint/typecheck/unit/E2E/build 검증

### 2.2 제외

- 로그인, 초대, 비밀번호, 회원가입
- 역할 기반 권한, 교사 배정 권한, 관리자 권한
- Supabase Auth, 역할 기반 권한, 교사별 접근 제한
- 여러 조직/교회 전환
- 서버 사진 업로드와 외부 파일 저장
- CSV 가져오기/내보내기
- 결제, 공개 SaaS 운영 기능

### 2.3 나중에 결정할 사항

- 공개 쓰기 상태를 언제 공유 코드 또는 Supabase Auth로 막을 것인가?
- 생일 연도가 필요한가, 월/일만으로 충분한가?
- 반이 늘어날 때 반 생성/수정 UI가 필요한가?
- 출석 상태가 `present/absent` 외에 지각, 공결까지 필요한가?

## 4. 핵심 흐름

### 4.1 출석 체크

1. 앱을 연다.
2. 홈 또는 하단 탭에서 출석을 누른다.
3. 아이별 출석과 큐티 완료를 체크하면 해당 아이 기록이 바로 저장된다.
4. 필요하면 이번 주 메모를 적는다.
5. 이번 주 메모 카드 안의 저장 버튼을 눌러 메모와 전도사님 공유 여부만 저장한다.
6. 아이별 저장 상태와 메모 저장 상태를 확인한다.

### 4.2 아이 추가

1. 아이들 탭을 연다.
2. 아이 추가 버튼을 누른다.
3. 모달에서 이름, 사진, 성별, 생년월일, 보호자, 주소, 이메일, 등록일, 반, 특이사항을 입력한다.
4. 저장하면 목록과 출석 화면에 반영된다.

### 4.3 통계 확인

1. 통계 탭을 연다.
2. 이번 주 출석 수, 이번 달 큐티 참여/완료 건수, 생일자를 확인한다.
3. 주간 출석, 월간 큐티, 월간 생일 카드를 눌러 상세 명단을 확인하고 복사한다.

## 5. 데이터 모델

### 5.1 Supabase 저장 모델

Supabase 정규화 업무 테이블을 앱 상태의 읽기/쓰기 기준으로 사용한다. 브라우저 `localStorage`는 앱 상태의 기준으로 사용하지 않는다.
이전 버전에서 생성된 localStorage 키(`seed-family-open-store-v1`)는 앱 시작 시 삭제한다.
초기 렌더링 중에는 빈 store와 로딩 상태를 사용하며, Supabase 정규화 테이블이 비어 있어도 테스트용 샘플 데이터를 자동 생성하거나 표시하지 않는다.

- `organizations`: 기본 조직, 부서, 시간대
- `teachers`: 선생님 프로필, 생일, 전화번호, 활성 상태
- `classes`: 반 이름과 담임 선생님 연결
- `children`: 아이 프로필, 성별, 선택적 생년월일, 반, 주소, 이메일, 등록일, 특이사항, 활성 상태
- `child_parents`: 아이별 보호자 관계, 이름, 전화번호
- `attendance_sessions`: 날짜별 출석 세션, 메모, 전도사님 공유 여부
- `attendance_records`: 세션별 아이 출석 상태와 큐티 완료 여부

출석 화면의 출석/큐티 저장은 iPhone Safari 같은 모바일 브라우저에서도 빠르게 끝나도록 현재 날짜의 `attendance_sessions` 1건을 보장한 뒤 해당 아이의 `attendance_records` 1건만 upsert한다. 이번 주 메모 저장은 `attendance_sessions`의 `note`, `share_with_pastor`, `saved_at`만 저장하고 `attendance_records`를 변경하지 않는다. 아이·반·선생님 관리 화면처럼 전체 store 동기화가 필요한 경우에만 전체 정규화 테이블 저장 경로를 사용한다.

기존 `family_open_app_state` 단일 JSON 테이블은 이관 원본과 백업 용도로 유지한다. `20260626000200_normalized_family_schema.sql` migration은 기존 JSON 데이터를 새 테이블로 복사한다.

### 5.2 개인정보 원칙

- 사용자가 명시적으로 요청한 아이 상세 정보와 보호자 연락처를 입력할 수 있다.
- 사진은 서버 파일 업로드 없이 브라우저에서 Data URL로 변환해 Supabase `teachers.photo_data_url`, `children.photo_data_url`에 저장한다.
- 휴대폰 사진처럼 500KB를 넘는 이미지는 브라우저에서 프로필용 크기로 자동 축소하고, 저장되는 Data URL은 500KB 이하로 제한한다.
- 브라우저가 디코딩할 수 없는 이미지이거나 축소 후에도 500KB를 넘으면 오류를 표시하고 저장하지 않는다.
- 사진 입력은 기본 파일 선택 문구를 노출하지 않고, 사진 영역 클릭으로 선택한다. 사진이 있으면 사진 클릭 시 수정/삭제 메뉴를 보여준다.
- 특이사항은 알러지 등 출석 관리에 필요한 짧은 내용만 적는 것을 전제로 한다.
- 아이 생년월일을 모르면 비워 둘 수 있으며, 출생연도만 아는 경우 연도만 저장한다. 생일 월/일이 없는 아이는 월간 생일자 통계에서 제외한다.
- Supabase migration이 적용되지 않았거나 네트워크 오류가 있으면 저장 실패 상태를 표시한다.

### 5.3 향후 보안 전환

공개 배포 또는 외부 사용자 접근 전에 새 `goal.md`로 다음을 다시 설계한다.

- 로그인 또는 공유 코드 방식
- 업무 테이블 migration
- RLS 또는 공유 키 기반 제한 정책
- 백업/복구, 감사 로그, 다중 기기 충돌 처리
- 다중 기기 충돌 처리

## 6. 화면 설계

### 6.1 홈 `/dashboard`

- 교회 이름과 부서
- 반 선택
- 반별 담임 선생님 표시
- 선택한 반의 출석 화면 이동

### 6.2 출석 `/attendance`

- 날짜 선택
- 반 선택: 반 이름과 담임 선생님 표시
- 아이 명단은 선택된 반에서는 가나다순, 전체 범위에서는 반별 정렬 후 가나다순으로 표시
- 저장되지 않은 새 세션의 아이별 출석 기본값은 미출석
- 아이 이름 앞 아바타 표시
- 아이별 출석 토글
- 아이별 큐티 완료 토글
- 아이 이름 옆 상세정보 아이콘
- 이번 주 메모
- 전도사님 공유 체크박스. 저장되지 않은 새 세션의 기본값은 unchecked다.
- 출석/큐티 토글은 누른 아이의 현재 날짜 기록만 즉시 저장한다.
- 아이별 row 안에 저장 중, 저장됨, 저장 실패 상태를 표시한다.
- 출석/큐티 저장 실패 시 체크한 값은 화면에 남기고 row 안의 재시도 버튼으로 같은 값을 다시 저장할 수 있어야 한다.
- 저장 버튼은 이번 주 메모 카드 안에 배치한다.
- 메모 저장 버튼 주변에 메모 변경 여부와 저장 실패 상태를 표시한다.
- 메모 저장 버튼은 이번 주 메모와 전도사님 공유 여부만 저장한다.
- 출석/큐티 저장 중에는 해당 아이 토글의 중복 조작을 막고, 메모 저장 중에는 메모 입력과 메모 저장 버튼의 중복 조작을 막는다.
- 모바일에서는 날짜 선택 입력이 카드 영역을 벗어나지 않아야 한다.

### 6.3 아이들 `/children`

- 아이 목록
- 반별 필터: 반 이름과 담임 선생님 표시
- 아이 이름 필터
- 정렬 선택: 가나다, 반별. 기본은 가나다순
- 아이 추가 모달
- 아이 카드 클릭 시 상세정보 보기/수정 모달
- 모바일 상세정보 모달은 짧은 필드를 2열로 배치할 수 있고, 날짜 입력은 다른 입력창 폭과 맞춰 카드 밖으로 넘치지 않아야 한다.
- 빈 상태
- 아이별 아바타, 이름, 반, 생년월일 또는 미입력 상태, 성별, 보호자 이름, 특이사항 표시

### 6.4 선생님 `/teachers`

- 선생님 등록 모달: 이름, 생일, 반, 전화번호, 사진
- 모바일 선생님 등록/수정 모달은 짧은 필드를 2열로 배치할 수 있다.
- 등록된 선생님 목록: 사진, 이름, 반, 생일, 전화번호 표시
- 등록된 선생님 카드를 클릭하면 상세/수정 모달이 열린다.

### 6.5 통계 `/reports`

- 이번 주 출석 인원
- 이번 달 큐티 참여 아동 수
- 이번 달 큐티 완료 건수
- 주간 출석 상세 명단 모달
- 월간 큐티 상세 명단 모달
- 월간 생일 상세 명단 모달
- 상세 명단 복사 버튼

### 6.6 설정 `/settings`

- 반 등록: 반 이름, 담임 선생님
- 등록된 반 목록: 반 이름, 담임 선생님 표시

## 7. UI·UX 기준

- 360px 너비부터 설계한다.
- 주요 터치 영역은 최소 44×44px, 출석 토글은 48px 이상을 권장한다.
- 입력 폰트는 16px 이상 사용한다.
- 하단 내비게이션은 safe area를 반영한다.
- 가로 스크롤을 만들지 않는다.
- 주요 페이지 상단에는 `저장됨` 상태 배지를 표시하지 않는다. 저장 전후 피드백은 각 작업 흐름 안의 변경 여부, 버튼 상태, 오류 안내로 제공한다.
- `DESIGN.md`의 밝고 친근한 분위기, `#58cc02` CTA, `#1cb0f6` 보조색, 12px radius를 사용한다.
- 색만으로 상태를 전달하지 않고 텍스트와 아이콘을 함께 사용한다.
- 출석/큐티 토글의 미선택 상태는 회색 계열로 표시하고, 선택 상태에서만 초록/파랑 강조색을 사용한다.
- 사진 없는 아이 기본 아바타는 성별에 따라 남자 파란 계열, 여자 분홍 계열, 미입력 초록 계열로 구분한다.

## 9. 테스트 전략

### 9.1 단위 테스트

- 날짜 계산
- 생일 월/일 검증
- 출석/큐티 통계 계산
- Supabase 정규화 저장 helper와 원격 상태 보정 helper

## 11. 2026-06-26 관리 기능 보강

- `/teachers`는 등록된 선생님 카드 클릭으로 상세/수정 모달 진입을 제공하고, 삭제는 수정 모달 안에서 제공한다.

- 공통 상단 저장 상태 배지는 제거되어 주요 페이지 헤더에는 `저장됨`을 표시하지 않는다.
- E2E는 Supabase 상태 초기화, 선생님 수정/삭제, 반 수정/삭제, 원격 저장 후 새로고침 유지를 검증한다.

## 12. 2026-06-26 미지정 등록 허용

- 아이 등록/수정 모달은 반을 선택하지 않아도 저장할 수 있다.
- 반 미지정 아이는 아이들 화면의 전체 목록에는 표시되고, 특정 반 출석 화면에는 자동 포함되지 않는다.
- 선생님 등록/수정 모달은 반을 선택하지 않아도 저장할 수 있다.
- 반 미지정 선생님은 어떤 반의 담임도 차지하지 않는다.
- 반 등록/수정은 담임 선생님을 선택하지 않아도 저장할 수 있다.
- 담임 미지정 반은 홈/출석/설정 화면에서 미지정 상태로 표시한다.
- Supabase 정규화 테이블 저장 방식은 유지하며 별도 migration은 추가하지 않는다.

## 13. 2026-06-26 삭제 confirm 및 모달 위치

- 아이, 선생님, 반 삭제는 목록 카드에서 직접 실행하지 않는다.
- 삭제 버튼은 해당 항목의 상세/수정 모달 안에 위치한다.
- 삭제 전에는 브라우저 기본 `window.confirm`으로 사용자 확인을 받는다.
- 아이 삭제는 출석 기록 참조 보존을 위해 `isActive: false`로 숨긴다.
- 선생님 삭제는 `isActive: false`로 숨기고, 해당 선생님이 담임인 반의 `teacherId`를 비운다.
- 반 삭제는 반을 제거하고, 그 반에 연결된 아이는 `classId: ""`로 변경해 반 미지정 상태로 남긴다.
- 모달 삭제 버튼은 저장/취소 액션과 분리된 위험 액션 스타일을 사용한다.

## 14. 2026-06-28 iPhone 저장 안정성 보강

- `/attendance` 출석/큐티 저장은 전체 store 저장 대신 현재 출석 날짜의 해당 아이 record만 Supabase에 저장한다.
- 출석/큐티 저장은 `attendance_sessions`를 upsert해 session id를 보장한 뒤 `attendance_records`의 해당 `session_id + child_id` 행만 upsert한다.
- 이번 주 메모 저장은 이번 주 메모 카드 안의 저장 버튼으로만 수행하며, `attendance_sessions`의 메모와 전도사님 공유 여부만 저장한다.
- 저장 중에는 같은 아이의 중복 토글과 메모 중복 제출을 막아 느린 모바일 네트워크에서 요청이 겹치지 않게 한다.
- 출석/큐티 저장 실패 시 row에 `저장 실패`를 표시하고 입력값을 유지해 같은 값으로 재시도할 수 있게 한다.
- 메모 저장 실패 시 `메모를 저장하지 못했습니다. 입력 내용은 남아 있습니다.`를 표시하고 입력값을 유지해 재시도할 수 있게 한다.
- 아이 아바타 사진은 `loading="lazy"`와 `decoding="async"`를 사용해 긴 명단에서 iPhone Safari 초기 렌더링 부담을 줄인다.

## 15. 2026-07-02 선생님 로그인, 관리자, 댓글형 메모

- 사이트 접속 시 현재 선생님을 알 수 없으면 전역 로그인 모달을 표시한다.
- 로그인 모달은 활성 선생님 목록과 비밀번호 입력을 제공하며, 초기 공통 비밀번호는 `1234`다.
- 로그인 성공 후 브라우저 `localStorage`에 현재 선생님 ID를 저장해 같은 브라우저에서는 재로그인하지 않는다.
- 선생님 데이터에는 `is_admin`을 저장한다. 기존 데이터에 관리자가 없으면 첫 활성 선생님을 초기 관리자로 지정한다.
- 선생님 상세 모달에서 관리자 여부를 지정할 수 있으며, 관리자 지정 UI는 관리자에게만 보인다.
- 이번 주 메모는 저장 시 기존 내용을 덮어쓰지 않고 `attendance_memos`에 댓글형 항목으로 추가한다.
- 메모는 날짜와 선택 반 기준으로 조회한다. 특정 반 선택 시 해당 반 메모만, 전체 선택 시 전체 메모를 표시한다.
- 메모 목록은 한 페이지 5개씩 페이지 처리한다.
- 기존 `전도사님 공유` 라벨은 `비밀`로 변경한다.
- 비밀 메모 내용은 작성자와 관리자만 볼 수 있으며, 권한이 없으면 비밀 처리된 문구만 표시한다.
- 특정 반의 비밀 메모는 해당 반 담임 선생님만 작성할 수 있다. 일반 메모는 다른 선생님도 작성할 수 있다.
- 현재 MVP는 공개 Supabase anon 쓰기 구조를 유지하므로, 이 기능의 권한 검증은 클라이언트와 DB 필드 기반이다. Supabase Auth/RLS 기반의 완전한 서버 권한 경계는 별도 작업으로 전환해야 한다.
