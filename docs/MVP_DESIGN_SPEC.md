## 1. 제품 개요

### 1.1 한 문장 설명

가족과 가까운 교회 구성원이 계정 없이 바로 열어 반 아이들의 주간 출석과 큐티 여부를 빠르게 기록하고, 간단한 생일·출석·큐티 현황을 확인하는 모바일 우선 웹앱이다.

### 1.2 이번 MVP의 성격

이번 MVP는 공개 SaaS가 아니라 **패밀리 오픈**이다.

- 로그인, 초대, 역할 권한, 결제는 만들지 않는다.
- 한 교회 또는 한 가족 규모의 사용을 전제로 한다.
- Supabase Postgres의 정규화 업무 테이블을 사용해 여러 기기에서 같은 데이터를 공유한다.
- 로그인 없는 패밀리 오픈이므로 publishable key 기반 공개 읽기/쓰기를 허용한다.
- 앱 상태는 브라우저 `localStorage`에 저장하지 않고 Supabase DB를 기준으로 읽고 쓴다.
- 앱 런타임은 테스트용 샘플 반/아이 데이터를 초기값이나 빈 DB fallback으로 사용하지 않는다.

### 1.3 성공 기준

- 휴대폰에서 3단계 이내로 출석 체크 화면에 도달한다.
- 아이별 출석과 큐티를 한 손으로 빠르게 바꿀 수 있다.
- 저장 여부가 명확하고 새로고침·기기 변경 후에도 Supabase에 저장된 데이터가 유지된다.
- 아이 상세 정보와 보호자 연락처는 Supabase 정규화 테이블에 저장된다.
- 모바일 390px 화면에서 가로 스크롤 없이 사용할 수 있다.

## 2. MVP 범위

### 2.1 Must Have

1. 로그인 없는 홈 진입
2. 기본 반과 아이 목록
3. 아이 추가
4. 반별 출석/결석 체크
5. 아이별 큐티 완료 체크
6. 간단한 주간 메모
7. Supabase 원격 저장과 새로고침 후 유지
8. 이번 주 출석 요약
9. 이번 달 큐티 참여 아동 수와 완료 건수
10. 이번 달 생일자 목록
11. 모바일 우선 하단 내비게이션
12. 기본 빈 상태, 저장 상태, 오류 방어
13. 아이 상세 정보 입력: 이름, 사진, 성별, 생년월일, 보호자 여러 명, 주소, 이메일, 등록일, 반, 특이사항
14. 아이 목록 반별/이름 필터와 가나다/반별 정렬
15. 주별·월별 막대그래프, 아바타/목록 상세 명단 모달과 명단 복사
16. 선생님 등록: 이름, 생일, 반, 전화번호, 사진
17. 설정 화면 반 등록: 반 이름과 담임 선생님 선택
18. 홈과 출석 반 선택에 담임 선생님 표시
19. 반별 댓글형 일반/비밀 메모
20. lint/typecheck/unit/E2E/build 검증

### 2.2 제외

- 로그인, 초대, 비밀번호, 회원가입
- 역할 기반 권한, 교사 배정 권한, 관리자 권한
- Supabase Auth, 역할 기반 권한, 교사별 접근 제한
- 여러 조직/교회 전환
- 서버 사진 업로드와 외부 파일 저장
- CSV 가져오기/내보내기
- 결제, 공개 SaaS 운영 기능

### 2.3 나중에 결정할 사항

- 공개 쓰기 상태를 언제 공유 코드 또는 Supabase Auth로 막을 것인가?
- 생일 연도가 필요한가, 월/일만으로 충분한가?
- 반이 늘어날 때 반 생성/수정 UI가 필요한가?
- 출석 상태가 `present/absent` 외에 지각, 공결까지 필요한가?

## 4. 핵심 흐름

### 4.1 출석 체크

1. 앱을 연다.
2. 홈 또는 하단 탭에서 출석을 누른다.
3. 아이별 출석과 큐티 완료를 체크하면 해당 아이 기록이 바로 저장된다.
4. 아이 카드의 한 줄 메모가 필요하면 입력 옆 저장 버튼으로 해당 날짜의 최신 메모를 저장한다.
5. `월간 현황`에서 선택 월의 모든 주일과 현재 반 아이별 출석·큐티 횟수를 확인하고, 카드를 펼쳐 지난 날짜를 바로 보완한다.
6. 필요하면 이번 주 메모를 적는다.
7. 이번 주 메모 카드 안의 저장 버튼을 눌러 일반 또는 비밀 댓글형 메모를 추가한다.
8. 아이별 저장 상태와 메모 저장 상태를 확인한다.

### 4.2 아이 추가

1. 아이들 탭을 연다.
2. 아이 추가 버튼을 누른다.
3. 모달에서 이름, 사진, 성별, 생년월일, 보호자, 주소, 이메일, 등록일, 반, 특이사항을 입력한다.
4. 저장하면 목록과 출석 화면에 반영된다.

기존 아이의 상세정보를 수정할 때 새 사진을 촬영하거나 앨범에서 선택하면 이미지 처리가 끝난 뒤 사진만 즉시 저장한다. 이름·보호자·반 등 다른 수정 중인 필드는 `수정 저장` 전까지 원격 DB에 반영하지 않는다. 신규 아이는 아직 DB 식별자가 없으므로 기존처럼 아이 전체를 저장할 때 사진을 함께 등록한다.

### 4.3 통계 확인

1. 통계 탭을 연다.
2. 각 카드의 `최근`에서 최근 4주 출석과 최근 4개월 큐티 완료자·생일자를 막대그래프로 확인한다.
3. `전체`로 바꾸면 2026년 7월부터 2027년 6월까지의 추이를 클릭되지 않는 라인그래프로 확인한다.
4. 최근 막대를 눌러 해당 아동의 아바타 grid를 보고, 필요하면 목록 보기로 전환하거나 명단을 복사한다.
5. 생일 미입력 명단에서 아이 상세정보를 열어 바로 보완한다.
6. 모든 반 선생님이 작성한 메모를 최신순 통합 목록으로 확인하고, 관리자는 확인 상태를 체크한다.

## 5. 데이터 모델

### 5.1 Supabase 저장 모델

Supabase 정규화 업무 테이블을 앱 상태의 읽기/쓰기 기준으로 사용한다. 브라우저 `localStorage`는 앱 상태의 기준으로 사용하지 않는다.
이전 버전에서 생성된 localStorage 키(`seed-family-open-store-v1`)는 앱 시작 시 삭제한다.
초기 렌더링 중에는 빈 store와 로딩 상태를 사용하며, Supabase 정규화 테이블이 비어 있어도 테스트용 샘플 데이터를 자동 생성하거나 표시하지 않는다.
전역 앱 상태는 단일 `FamilyOpenStoreProvider`에서 한 번 로드하고, 로그인 모달과 각 화면은 같은 store context를 공유한다.

- `organizations`: 기본 조직, 부서, 시간대
- `teachers`: 선생님 프로필, 생일, 전화번호, 활성 상태
- `classes`: 반 이름과 담임 선생님 연결
- `children`: 아이 프로필, 성별, 선택적 생년월일, 반, 주소, 이메일, 등록일, 특이사항, 활성 상태
- `child_parents`: 아이별 보호자 관계, 이름, 전화번호
- `attendance_sessions`: 날짜별 출석 세션과 이전 버전 호환용 메모 필드
- `attendance_records`: 세션별 아이 출석 상태, 큐티 완료 여부, 이력 없이 최신값만 유지하는 날짜별 아이 한 줄 메모
- `attendance_memos`: 날짜·반·작성자별 댓글형 일반/비밀 메모와 관리자 확인 시각·확인자

출석 화면의 출석/큐티 저장은 iPhone Safari 같은 모바일 브라우저에서도 빠르게 끝나도록 현재 날짜의 `attendance_sessions` 1건을 보장한 뒤 해당 아이의 `attendance_records` 1건만 upsert한다. 이번 주 메모는 `attendance_memos` 전용 insert 경로로 댓글형 항목을 추가하고 `attendance_records`를 변경하지 않는다. 아이·반·선생님 관리 화면처럼 전체 store 동기화가 필요한 경우에만 전체 정규화 테이블 저장 경로를 사용한다.

아이 한 줄 메모는 100자 이하이며 `attendance_records.note` 최신값만 덮어쓴다. 메모 전용 upsert는 `note`만 변경하고 출석·큐티 열을 보내지 않으며, 출석·큐티 개별 upsert도 `note`를 보내지 않아 모바일 네트워크 응답 순서가 엇갈려도 서로의 최신값을 덮어쓰지 않는다.

아이·반·선생님 관리 화면의 전체 store 저장은 현재 브라우저가 알고 있는 메모만 ID 기준 upsert하고 조직 메모를 전량 삭제하지 않아, 다른 교사가 동시에 추가한 메모를 보존한다.

기존 `family_open_app_state` 단일 JSON 테이블은 이관 원본과 백업 용도로 유지한다. `20260626000200_normalized_family_schema.sql` migration은 기존 JSON 데이터를 새 테이블로 복사한다.

### 5.2 개인정보 원칙

- 사용자가 명시적으로 요청한 아이 상세 정보와 보호자 연락처를 입력할 수 있다.
- 사진은 서버 파일 업로드 없이 브라우저에서 Data URL로 변환해 Supabase `teachers.photo_data_url`, `children.photo_data_url`에 저장한다.
- 휴대폰 사진처럼 160KB를 넘는 이미지는 브라우저에서 프로필용 크기로 자동 축소하고, 저장되는 Data URL은 160KB 이하로 제한한다.
- 브라우저가 디코딩할 수 없는 이미지이거나 축소 후에도 160KB를 넘으면 오류를 표시하고 저장하지 않는다.
- 사진 입력은 기본 파일 선택 문구와 설명성 제목을 노출하지 않고 미리보기 옆에 촬영·앨범 동작을 제공한다. 저장된 사진을 누르면 큰보기 모달이 열리고 삭제는 큰보기 안에서만 제공한다.
- 아이와 선생님 사진 입력은 모바일에서 실제 native file input을 직접 활성화하는 `사진 찍기`와 `앨범에서 선택`을 분리해 제공한다. `사진 찍기`는 후면 카메라 capture를 요청한다.
- 기존 아이의 새 사진은 처리 완료 즉시 `organization_id + child id`로 제한된 `children.photo_data_url` 단독 PATCH로 저장한다. 전체 store 저장 경로를 사용하지 않으며, 성공한 뒤에만 전역 store의 해당 아이 사진을 갱신한다.
- 사진 자동 저장 중에는 중복 촬영·선택과 폼 제출을 막고, 성공 상태를 모달 안에서 알린다. 실패하면 선택 사진과 다른 폼 draft를 유지하고 사진 전용 재시도를 제공한다. 닫기와 취소는 네트워크 지연 중에도 사용할 수 있다.
- 기존 사진 삭제는 되돌리기 어려운 동작이므로 자동 저장하지 않고 기존 `수정 저장` 흐름에서 반영한다. 선생님 사진과 신규 아이 사진도 기존 폼 저장 흐름을 유지한다.
- MIME 정보가 비어 있는 모바일 이미지도 알려진 이미지 확장자 또는 브라우저 디코딩으로 확인하고, HEIC/HEIF와 MIME 미상 이미지는 작은 파일이어도 JPEG 변환 경로를 사용한다. 디코딩할 수 없는 형식은 원본을 저장하지 않고 재선택 안내를 표시한다.
- 특이사항은 알러지 등 출석 관리에 필요한 짧은 내용만 적는 것을 전제로 한다.
- 아이 생년월일을 모르면 비워 둘 수 있으며, 출생연도만 아는 경우 연도만 저장한다. 생일 월/일이 없는 아이는 월간 생일자 통계에서 제외한다.
- Supabase migration이 적용되지 않았거나 네트워크 오류가 있으면 저장 실패 상태를 표시한다.

## 6. 화면 설계

### 6.1 홈 `/dashboard`

- 교회 이름과 부서
- 반 선택
- 반별 담임 선생님 표시
- 선택한 반의 출석 화면 이동

### 6.2 출석 `/attendance`

- 날짜 선택
- `일별 체크`와 `월간 현황` 전환
- 반 선택: 반 이름과 담임 선생님 표시
- 현재 선택된 반은 초록 테두리 select 하나로 명확히 표시하고 별도 반복 배지는 두지 않는다.
- 아이 명단은 선택된 반에서는 가나다순, 전체 범위에서는 반별 정렬 후 가나다순으로 표시
- 저장되지 않은 새 세션의 아이별 출석 기본값은 미출석
- 아이 이름 앞 아바타 표시
- 아이별 출석 토글
- 아이별 큐티 완료 토글
- 아이별 100자 이하 한 줄 메모와 명시적 저장 버튼. 이 메모는 선택 날짜별 최신값만 유지하며 이번 주 메모와 별개다.
- 아이 이름 옆 상세정보 아이콘
- 이번 주 메모
- 비밀 체크박스. 새 메모의 기본값은 unchecked이며, 비밀 메모는 해당 반 담임만 작성할 수 있다.
- 출석/큐티 토글은 누른 아이의 현재 날짜 기록만 즉시 저장한다.
- 아이별 row 안에 저장 중, 저장됨, 저장 실패 상태를 표시한다.
- 출석/큐티 저장 실패 시 체크한 값은 화면에 남기고 row 안의 재시도 버튼으로 같은 값을 다시 저장할 수 있어야 한다.
- 저장 버튼은 이번 주 메모 카드 안에 배치한다.
- 메모 저장 버튼 주변에 메모 변경 여부와 저장 실패 상태를 표시한다.
- 메모 저장 버튼은 입력한 내용을 기존 메모를 덮어쓰지 않는 댓글형 항목으로 추가한다.
- 전체 반 선택에서는 메모 작성과 메모 목록을 표시하지 않는다. 반별 작성 중 메모는 날짜와 반을 함께 기준으로 분리해 다른 반으로 따라가지 않게 한다.
- 출석/큐티 저장 중에는 해당 아이 토글의 중복 조작을 막고, 메모 저장 중에는 메모 입력과 메모 저장 버튼의 중복 조작을 막는다.
- 모바일에서는 날짜 선택 입력이 카드 영역을 벗어나지 않아야 한다.
- 월간 현황은 선택 월의 모든 주일을 기록 유무와 함께 2열 요약으로 표시하고, 날짜를 누르면 해당 일별 체크로 이동한다.
- 월간 현황은 현재 선택 반의 활성 아이를 기준으로 아이별 출석 횟수와 큐티 횟수를 표시한다. 아이 카드를 펼치면 각 주일의 출석·큐티를 즉시 수정할 수 있다.
- 월간 집계는 현재 반 소속 기준이다. 현재 스키마에는 반 이동 당시의 반 snapshot이 없어 과거 소속 반 기준 집계는 제공하지 않는다.

### 6.3 아이들 `/children`

- 반별 필터: 반 이름과 담임 선생님 표시
- 아이 이름 필터

### 6.5 통계 `/reports`

- 주간 출석 기준일과 생일/큐티 월 필터는 표시하지 않는다.

- `전체`는 2026년 7월~2027년 6월의 출석 주별·큐티 월별·생일 월별 추이를 클릭되지 않는 라인그래프로 표시한다.
- 최근 막대 클릭 시 해당 출석자·큐티 완료자·생일자의 아바타 grid 상세 명단 모달을 연다.
- 아바타 보기에는 사진 또는 기본 아바타만 표시하고, 목록 보기에는 이름·반·세부 정보를 표시한다. 상세 명단에는 보기 토글과 명단 복사 버튼을 제공한다.
- 생일 미입력 활성 아이 수와 목록을 제공하고, 목록에서 아이 상세/수정 모달을 바로 연다.
- 모든 날짜와 반의 선생님 메모를 최신순 5개씩 표시하는 통합 목록
- 통계 통합 목록의 비밀 메모 본문은 관리자만 보고, 다른 선생님에게는 비밀 처리 문구를 표시한다.

## 14. 2026-06-28 iPhone 저장 안정성 보강

- `/attendance` 출석/큐티 저장은 전체 store 저장 대신 현재 출석 날짜의 해당 아이 record만 Supabase에 저장한다.
- 출석/큐티 저장은 `attendance_sessions`를 upsert해 session id를 보장한 뒤 `attendance_records`의 해당 `session_id + child_id` 행만 upsert한다.
- 이번 주 메모 저장은 이번 주 메모 카드 안의 저장 버튼으로만 수행하며, `attendance_memos`에 댓글형 항목을 추가하고 출석 record는 변경하지 않는다.
- 저장 중에는 같은 아이의 중복 토글과 메모 중복 제출을 막아 느린 모바일 네트워크에서 요청이 겹치지 않게 한다.
- 출석/큐티 저장 실패 시 row에 `저장 실패`를 표시하고 입력값을 유지해 같은 값으로 재시도할 수 있게 한다.
- 메모 저장 실패 시 `메모를 저장하지 못했습니다. 입력 내용은 남아 있습니다.`를 표시하고 입력값을 유지해 재시도할 수 있게 한다.
- 아이 아바타 사진은 `loading="lazy"`와 `decoding="async"`를 사용해 긴 명단에서 iPhone Safari 초기 렌더링 부담을 줄인다.

## 15. 2026-07-02 선생님 로그인, 관리자, 댓글형 메모

- 사이트 접속 시 현재 선생님을 알 수 없으면 전역 로그인 모달을 표시한다.
- 로그인 모달은 활성 선생님 목록을 가나다순으로 표시한다. 비밀번호 입력은 비활성화하고, 선생님 선택만으로 로그인한다.
- 로그인 성공 후 브라우저 `localStorage`에 현재 선생님 ID를 저장해 같은 브라우저에서는 재로그인하지 않는다.
- 선생님 데이터에는 `is_admin`을 저장한다. 기존 데이터에 관리자가 없으면 첫 활성 선생님을 초기 관리자로 지정한다.
- 선생님 상세 모달에서 관리자 여부를 지정할 수 있으며, 관리자 지정 UI는 관리자에게만 보인다.
- 이번 주 메모는 저장 시 기존 내용을 덮어쓰지 않고 `attendance_memos`에 댓글형 항목으로 추가한다.
- 출석 화면 메모는 날짜와 선택 반 기준으로 조회한다. 특정 반 선택 시 해당 반 메모만 표시하고, 전체 반 선택에서는 메모 작성과 목록을 모두 숨긴다.
- 출석 화면과 통계 통합 메모 목록은 한 페이지 5개씩 페이지 처리한다.
- 기존 `전도사님 공유` 라벨은 `비밀`로 변경한다.
- 출석 화면의 비밀 메모 내용은 작성자와 관리자만 볼 수 있으며, 통계 통합 목록에서는 관리자만 볼 수 있다. 권한이 없으면 비밀 처리된 문구만 표시한다.
- 특정 반의 비밀 메모는 해당 반 담임 선생님만 작성할 수 있다. 일반 메모는 다른 선생님도 작성할 수 있다.
- 통계 통합 목록의 메모 확인은 관리자만 변경하며 `acknowledged_at`, `acknowledged_by_teacher_id`에 저장한다. 현재 앱의 관리자 검증은 클라이언트 로그인과 DB 필드 기반이다.
- 현재 MVP는 공개 Supabase anon 쓰기 구조를 유지하므로, 이 기능의 권한 검증은 클라이언트와 DB 필드 기반이다. Supabase Auth/RLS 기반의 완전한 서버 권한 경계는 별도 작업으로 전환해야 한다.

## 16. 2026-07-15 아이 사진 즉시 저장과 모바일 UX 보강

- 기존 아이의 촬영·앨범 선택 사진은 처리 완료 후 해당 아이의 `photo_data_url` 한 필드만 즉시 저장한다.
- 사진 저장은 실제 갱신 행을 확인해 대상이 없거나 RLS로 가려진 0행 갱신을 성공으로 표시하지 않는다.
- DB 원문 오류를 화면에 노출하지 않고 선택 사진과 다른 폼 입력을 유지한 채 재시도 방법을 안내한다.
- 아이 상세 모달에는 focus trap, 초기 focus, focus 복원과 iPhone safe-area 하단 여백을 적용한다.
- 출석 상세·오류 재시도·메모 페이지 이동 버튼은 최소 44px 터치 높이를 사용하고, 큐티 선택 상태는 원본 하늘색 위 어두운 글자로 대비를 보정한다.
- 아이·선생님·반 목록은 초기 로딩과 실제 빈 상태를 구분한다.
- 원격 Supabase를 초기화하는 `app-shell` E2E는 `E2E_ALLOW_REMOTE_RESET=1`을 명시한 전용 테스트 환경에서만 실행하고, 기본 검증은 REST mock E2E를 사용한다.

## 17. 2026-07-19 월간 출석 확인과 아이별 날짜 메모

- `/attendance`에 `일별 체크`와 `월간 현황` 전환을 제공한다.
- 월간 현황 진입 시 전체 반이 선택돼 있고 현재 선생님에게 담임 반이 있으면 그 반을 우선 선택한다.
- 선택 월의 모든 주일을 표시해 세션이 없는 날짜도 `기록 없음`으로 찾을 수 있게 한다.
- 주일별 요약을 누르면 해당 날짜의 일별 체크로 이동하고, 아이별 카드를 펼치면 월간 화면 안에서도 출석·큐티를 즉시 수정할 수 있다.
- 아이별 월간 횟수는 현재 선택 반에 소속된 활성 아이 기준으로 계산한다.
- 일별 아이 카드에는 100자 이하 한 줄 메모와 독립 저장 상태를 제공한다. 이 값은 `attendance_records.note` 최신값만 유지하며 이번 주 댓글형 메모와 분리한다.
- 아이 메모 저장과 출석·큐티 저장은 서로의 열을 갱신하지 않는 별도 upsert로 처리한다.
- 통계 상세의 아바타 보기는 원형 아바타만 표시하고 사각 카드 래퍼를 두지 않는다.
- 큐티 완료자 상세 요약에서는 월 총 완료 횟수를 제거하되 목록 보기와 복사 명단의 아이별 완료 횟수는 유지한다.
